# Multi-agent protocol

Когда задача затрагивает `spec/` + `formal/` + код, использовать параллельную работу нескольких специализированных агентов.

## Роли

- Spec Analyst: извлекает требования из `spec/` и формирует поведенческий контракт.
- Formal/TLA+ Analyst: определяет изменения в `formal/` и какие TLC-тесты нужны.
- Formal Engineer: редактирует `formal/**` (семантика и тесты TLC) и прогоняет `make test_*`.
- Implementation Engineer: вносит минимальные правки в код по контракту.
- Test Engineer: пишет/правит тесты (TLA+ и/или Python) и гоняет проверки.

## Правила синхронизации

- Исследования/поиск можно делать параллельно.
- Один файл редактирует только один агент (в идеале — главный агент).
- Если spec и formal расходятся: приоритет у `spec/`, затем привести `formal/` в соответствие.
- Любая новая логика обязана иметь проверку: тест должен падать до фикса и проходить после.
