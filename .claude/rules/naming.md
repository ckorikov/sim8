# Naming Conventions

## Доменные сокращения

Эти имена зафиксированы в кодовой базе — не изобретать синонимы.

### Архитектура и ISA
- `cpu` — абстрактная модель CPU
- `fpu` — Floating Point Unit
- `vu` — Vector Unit
- `mem` — память
- `arch` — версия архитектуры (int: 1/2/3)
- `instr` — инструкция; **никогда** `insn`. Инструкция = опкод + операнды.
- `mnemonic` — строковое имя инструкции (`"MOV"`, `"FADD"`)
- `op` — opcode (числовой код инструкции)
- `operands` — список операндов
- `format` — формат operands инструкции: список `OpType` в их порядке (не signature)
- `cost` — стоимость инструкции в циклах
- `InstrDef` — тип/класс определения инструкции; **никогда** `OpDef`, `InsnDef`

### Парсер и ассемблер
- `loc` — source location `(filename, line_no)`
- `prep` — `PreprocessResult`
- `state` — имя для локального mutable state-объекта внутри pass/phase; не использовать сокращение `st` и не использовать для immutable объектов
- `encoded` — результирующий список байт одной инструкции
- `patch` — один fixup-дескриптор для отложённого разрешения ссылки на метку
- `patches` — список fixup-дескрипторов

### Форматы и регистры (FPU/VU домен)
- `fmt` — код формата (`FP_FMT_*`, `VU_FMT_*`)
- `phys` — индекс физического FP-регистра
- `pos` — позиция внутри формата
- `rm` — rounding mode
- `except` — объект FP-исключений
- `fpm` — floating point mode
- `fpm_enc` — закодированное значение FP Modifier (результат `_encode_fpm`); **никогда** `fpm_byte`, `fpm_bytes`
- `vum` — VU mode (концептуальный термин для поля mode)
- `vfm` — VU Format+Mode byte (1-байтовый VU-операнд, кодирует fmt + mode); локальная переменная → `vfm_enc` (Python) / `vfmEnc` (JS)

### Классы ошибок (строго по уровню)
- `AsmSyntaxError` — низкоуровневая ошибка ассемблерного синтаксиса: токены, литералы, операнды, суффиксы
- `ParseError` — парсер
- `PreprocessError` — препроцессор
- `AssemblerError` — ошибка этапа сборки/кодогенерации после успешного парсинга; использовать для семантики сборки, адресации, fixup и эмиссии байт

## Форматы имён

### Функции
- snake_case (Python) / camelCase (JS)
- Приватные — префикс `_`

### Классы
- PascalCase везде
- Ошибки — суффикс `Error`
- Операнды — префикс `Op` (Python)
- Определение инструкции — `InstrDef`

### Константы
- SCREAMING_SNAKE везде: `FP_FMT_F`, `VU_MODE_VV`, `ERR_DIV_ZERO`, `PAGE_SIZE`

## Семейства файлов

### Файлы (Python)
- Публичные модули: `codegen.py`, `parser.py`
- Приватные хелперы: `_codegen_core.py`, `_parser_types.py`

### Файлы (JavaScript)
- Публичные: `asm-parse.js`, `asm.js`, `isa.js`
- Приватные хелперы: `_asm-parse-ops.js`
- CPU handlers: `core-handlers-int.js`, `core-handlers-fp.js`, `core-handlers-vu.js`

### Тесты
- Файлы: `test_*.py`
- Spec-driven методы: `test_NNN_description` (NNN = ID из спецификации)

## Допустимые исторические/специальные исключения

- Всегда использовать сокращения из словаря выше; полные формы и синонимы в новых именах запрещены.
- Исключения допустимы только если имя уже закреплено внешним API, форматом, стандартом или историческим публичным интерфейсом.
- Любое исключение должно быть явно задокументировано в этом разделе; недокументированное исключение считать нарушением.
- Новые исключения не вводить без прямой необходимости.

## Термы (префиксы, инфиксы, суффиксы) функций

- `_encode_` — typed data → bytes
- `_decode_` — bytes → typed data
- `_try_` — парсинг; возвращает `None` при неудаче
- `_parse_` — детерминированный парсинг
- `_lookup_` — поиск
- `_validate_` — проверка ограничений; бросает при нарушении
- `_build_` — конструирует сложный объект
- `_resolve_` — разрешает имя/суффикс в конкретное значение
- `_handle_` — обработчик инструкции CPU

## Термы (префиксы, инфиксы, суффиксы) модулей

- `_parser_` — внутренние модули парсера
- `_codegen_` — внутренние модули кодогенератора
- `_asm_`  — модули ассемблера
